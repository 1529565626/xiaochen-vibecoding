> 请逐项确认默认值，在「你的调整值」栏填写期望的值，留空则使用默认值。填写完成后告诉我，我将以此为基准生成项目代码。

---

## 1. 服务基础配置

| 参数 | 默认值 | 说明 | 你的调整值 |
|------|--------|------|------------|
| `server.port` | `3000` | HTTP 服务端口，避免与常见服务冲突（80/8080 常被占用） | |
| `server.host` | `0.0.0.0` | 绑定地址，`0.0.0.0` 表示监听所有网卡（局域网可访问），`127.0.0.1` 仅本机 | |
| `server.autoOpen` | `true` | 启动后是否自动打开管理端页面 | |

---

## 2. 存储路径配置

| 参数 | 默认值 | 说明 | 你的调整值 |
|------|--------|------|------------|
| `storage.downloadDir` | `./files/download` | 文件下载根目录（客户端可浏览和下载的文件在这里） | E://my-netdisk/files/download |
| `storage.uploadDir` | `./files/upload` | 上传临时目录（分片上传的临时文件存放处，合并后移入 downloadDir） | E://my-netdisk/files/upload |
| `storage.tempDir` | `./files/temp` | 通用临时目录（分片合并等中间操作使用） | E://my-netdisk/files/temp |

> 以上路径支持相对路径（相对于 exe 所在目录）和绝对路径（如 `D:\my-files`）。  
> **注意**：三个目录必须不同，保证上传和下载在页面层面隔离。

---

## 3. 上传配置

| 参数 | 默认值 | 说明 | 你的调整值 |
|------|--------|------|------------|
| `upload.maxFileSize` | `10737418240` | 单文件最大体积（字节）默认 10GB | |
| `upload.maxBatchSize` | `21474836480` | 单次批量上传最大总量（字节）默认 20GB | |
| `upload.chunkSize` | `10485760` | 分片大小（字节）默认 10MB。网络不稳定可调小，网络好可调大 | |
| `upload.maxConcurrent` | `3` | 并发上传分片数。电脑配置偏低（4G 内存）建议改为 `2` | |
| `upload.mergeTimeout` | `600000` | 分片合并超时时间（毫秒），默认 10 分钟。超大文件合并耗时可调大 | |
| `upload.cleanupTemp` | `true` | 上传完成（含失败）后是否自动清理临时分片文件 | |

## 5. 管理端配置

| 参数 | 默认值 | 说明 | 你的调整值 |
|------|--------|------|------------|
| `admin.allowRemote` | `false` | 是否允许非本机访问管理端 API。`true` 时局域网其他设备也可访问管理功能 | |
| `admin.username` | `""` | 管理端登录用户名。留空表示不启用认证 | |
| `admin.password` | `""` | 管理端登录密码。留空表示不启用认证 | |

> 如果未来有多人使用场景需要管理员密码保护，在此填写用户名和密码。MVP 阶段建议留空。
