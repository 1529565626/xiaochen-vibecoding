# 需求规格报告 · 局域网文件管理服务（my-net-disk）

> 版本：v1.1 · 日期：2026-05-21 · 状态：已实现

---

## 1. 项目概述

### 1.1 项目定位

搭建一个运行在 Windows 电脑上的局域网文件管理服务。服务端启动后，同一局域网内的任意设备（PC、手机、平板）通过浏览器即可访问、上传、下载文件。本质是一个轻量级的私有 NAS Web 系统。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 文件共享 | 局域网内多设备共享文件，无需 U 盘或微信传输 |
| 大文件传输 | 10G 级单文件传输，不受即时通讯工具的文件大小限制 |
| 移动端访问 | 手机扫码即可浏览/上传/下载电脑上的文件 |
| 临时网盘 | 快速搭建，用完即关，无需注册账号 |

### 1.3 使用流程

```
桌面端双击启动 → 自动打开管理端页面 → 手机扫码访问客户端 → 浏览/上传/下载文件
```

## 3. 技术架构

### 3.1 技术选型

```
┌─────────────────────────────────────────┐
│              浏览器 (PC / Mobile)         │
│          Vue 3 + Vite + Ant Design Vue   │
├─────────────────────────────────────────┤
│                REST API                   │
├─────────────────────────────────────────┤
│           Node.js + Fastify              │
│   ┌──────┬──────┬──────┬──────────┐     │
│   │上传  │下载  │管理  │配置      │     │
│   │引擎  │引擎  │路由  │路由      │     │
│   └──────┴──────┴──────┴──────────┘     │
├─────────────────────────────────────────┤
│   SQLite (sql.js)          │  本地文件系统  │
└─────────────────────────────────────────┘
```

| 层级 | 选型 | 理由 |
|------|------|------|
| 后端框架 | **Fastify** | 流式文件处理，原生请求校验，大文件场景优势明显 |
| 前端框架 | **Vue 3 + Vite** | 生态成熟，Hash 路由适配静态部署 |
| UI 组件库 | **Ant Design Vue 4** | 表格、上传、进度条等组件开箱即用 |
| 数据库 | **SQLite (sql.js)** | 纯 JS 实现，无编译依赖，零配置部署 |
| 文件传输 | **分片上传 + Stream 下载** | 保证 10G+ 文件稳定传输 |
| 二维码 | **qrcode** | 服务端生成 SVG，终端和页面均可渲染 |
| 打包 | **esbuild 打包 + 目录分发** | 用户无需安装 Node.js |

### 3.2 关键设计决策

- **sql.js 替代 better-sqlite3**：避免 C++ 编译依赖，sql.js 为纯 WASM 实现，API 兼容
- **上传=uploadDir，下载=downloadDir**：完全隔离，`root` 参数贯穿所有 API
- **Fastify `routerOptions.maxParamLength`**：支持长 UUID 参数
- **Hash 路由**：`createWebHashHistory` 适配静态文件部署，无需服务端路由 fallback

## 4. API 设计

### 4.1 文件操作 API

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| `GET` | `/api/files` | `?path=/&root=download\|upload` | 获取目录文件列表 |
| `GET` | `/api/files/info` | `?path=/a.mp4&root=download\|upload` | 获取文件信息 |
| `POST` | `/api/upload/init` | `{filename, totalSize, targetPath?, root?}` | 初始化分片上传 |
| `POST` | `/api/upload/chunk` | multipart `{uploadId, chunkIndex, file}` | 上传单个分片 |
| `GET` | `/api/upload/status/:uploadId` | — | 已上传分片列表（用于断点续传） |
| `POST` | `/api/upload/complete/:uploadId` | — | 合并分片为完整文件 |
| `GET` | `/api/upload/tasks` | — | 所有上传任务状态 |
| `GET` | `/api/download` | `?path=/a.mp4&root=download\|upload` | 流式下载 |

### 4.2 管理端 API

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| `POST` | `/api/admin/folder` | `{path, name, root}` | 新建文件夹 |
| `PUT` | `/api/admin/folder` | `{path, oldName, newName, root}` | 重命名文件夹 |
| `DELETE` | `/api/admin/folder` | `{path, name, root}` | 删除文件夹 |
| `DELETE` | `/api/admin/file` | `{path, root}` | 删除文件 |
| `GET` | `/api/admin/config` | — | 获取当前配置 |
| `PUT` | `/api/admin/config` | 部分配置 JSON | 更新并持久化配置 |
| `GET` | `/api/admin/status` | — | 服务运行状态 |
| `GET` | `/api/admin/qrcode` | — | 局域网访问二维码（SVG） |
| `POST` | `/api/admin/open-folder` | `{path?, root}` | 资源管理器打开目录 |

### 4.3 分片上传流程

```
客户端                         服务端
  │                              │
  │── POST /upload/init ────────→│  创建 uploadId，返回 chunkSize / chunkCount
  │   {filename, totalSize,root} │
  │                              │
  │── GET /upload/status/:id ───→│  返回已上传分片索引（断点续传）
  │                              │
  │── POST /upload/chunk ───────→│  逐个发送分片（并发串行混合）
  │   multipart {id,idx,file}    │  每个分片写入 temp/chunks/{id}/{idx}
  │                              │
  │── POST /upload/complete/:id →│  校验所有分片 → Stream 合并 → 清理临时文件
  │                              │
```

### 4.4 安全规则

- 所有路径参数做 `normalize` + `..` 检测，拒绝目录穿越
- 管理端 API 仅本机可访问（`admin.allowRemote` 控制）
- 所有文件操作限定在配置的白名单目录内
- 文件名 sanitize：移除 `<>:"|?*\` 字符

## 6. 数据模型

### 6.1 上传任务表

```sql
CREATE TABLE uploads (
  id          TEXT PRIMARY KEY,
  filename    TEXT NOT NULL,
  target_path TEXT NOT NULL,
  total_size  INTEGER NOT NULL,
  chunk_size  INTEGER NOT NULL,
  chunk_count INTEGER NOT NULL,
  status      TEXT DEFAULT 'uploading',  -- uploading | merging | done | failed
  created_at  TEXT DEFAULT (datetime('now')),
  updated_at  TEXT DEFAULT (datetime('now'))
);
```

> 文件列表直接从文件系统读取。数据库仅记录上传任务状态。

## 8. 部署与打包

### 8.1 开发模式

```bash
cd server && npm run dev    # 后端 http://localhost:3000
cd web && npm run dev       # 前端 http://localhost:5173（代理到后端）
```

### 8.2 生产分发

```
release/
├── my-net-disk.exe / start.bat     # 启动入口
├── server.js                       # 打包后的服务端
├── node_modules/                   # 运行时依赖（或内嵌）
└── web/                            # 前端构建产物
```

- 最终用户无需安装任何依赖
- 双击启动，自动打开管理端

### 8.3 服务生命周期

```
启动 → 检测 IP → 检查路径权限 → 监听端口 → 打开管理端 → 显示二维码 → Ctrl+C 退出
```
